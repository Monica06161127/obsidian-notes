# 📝 部署最小智能合约

> 在 Sepolia 测试网部署 SimpleStorage 合约

## 合约信息

- **合约名称**：SimpleStorage
- **合约地址**：`0x265e61c8422D9dE1de2C45b3A659619E16C056eD`
- **部署网络**：Sepolia 测试网
- **创建者**：`0x7F1a...524DB`（我的钱包）
- **Etherscan**：[查看合约](https://sepolia.etherscan.io/address/0x265e61c8422D9dE1de2C45b3A659619E16C056eD)

## 合约功能

只有一个功能：存储和读取一个数字。

```solidity
function set(uint256 _number) public {  // 写入数字（需要签名+Gas）
    storedNumber = _number;
}

function get() public view returns (uint256) {  // 读取数字（免费）
    return storedNumber;
}
```

## 写入 vs 读取

| 操作 | 需要签名？ | 需要 Gas？ | 改变链上状态？ |
|------|-----------|-----------|--------------|
| `set(42)` | ✅ 是 | ✅ 是 | ✅ 是 |
| `get()` | ❌ 否 | ❌ 否 | ❌ 否 |

**核心理解**：写入 = 改变世界状态 = 需要共识 = 需要成本；读取 = 查看状态 = 免费。

## 学到的知识

1. 智能合约 = 部署在区块链上的自动执行程序
2. 部署 = 把程序放到区块链上，获得一个地址
3. 写入操作 = 改变链上状态 = 霗要签名 + Gas
4. 读取操作 = 查看链上状态 = 免费
5. 交易哈希 = 每笔链上操作的唯一标识
6. 区块浏览器 = 查看所有链上操作的工具
