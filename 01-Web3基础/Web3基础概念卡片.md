# 🔗 Web3 基础概念卡片

> 用生活类比帮助自己理解链上世界的底层逻辑

## 1. Account（账户）

**一句话**：Web3 的账户就是"链上身份"——所有操作都绑定在你的账户上，就像银行账户绑定资金流水。

**例子**：我的测试网账户 `0x7F1a...524DB` 是我在 Sepolia 链上的身份。所有交易都记录在这个地址下，全世界都能看到。

**误区**：Web3 账户和 Web2 完全不同——没有客服、没有找回密码、没有"忘记密码"按钮。**你是唯一的负责人。**

## 2. Address（地址）

**一句话**：地址就是你账户的"银行账号"——别人给你转账需要知道你的地址。

**例子**：`0x7F1a8ddbdECCA0945c392297e97CFe4694A524DB` 是我的地址，42 位十六进制字符，可在区块浏览器查所有记录。

**安全提醒**：地址可以公开分享，但 **地址 ≠ 控制权**。知道地址不能动钱，就像知道银行卡号不能取钱。

## 3. Wallet（钱包）

**一句话**：钱包不是"装钱的容器"，而是"管理私钥的工具"——币在链上，钱包只是保管钥匙。

**例子**：MetaMask 帮我管理私钥、签名交易、连接 DApp。就像密码管理器——钱在银行里，它只是帮你记住密码。

**误区**：删了 MetaMask 但有助记词，重新导入就能恢复所有资产——资产从来不"在"钱包里。

## 4. Seed Phrase（助记词）

**一句话**：12 或 24 个英文单词，是账户的"终极备份"——谁拥有它，谁就拥有所有资产。

**例子**：创建 MetaMask 时给的 12 个单词，能推导出所有私钥和地址。

**⚠️ 安全红线**：
- 永远不要截图、存手机、发给任何人
- 手抄纸上，存安全地方
- 任何让你"输入助记词验证"的网站都是诈骗

## 5. Private Key（私钥）

**一句话**：账户的"密码"——用它签名就能动用资产，丢了或泄露就完了。

**例子**：MetaMask 点"确认交易"就是用私钥签名，就像在支票上签字。

**⚠️ 安全红线**：
- 不要存在联网设备上
- 不要在任何网站输入
- 有人问你要私钥 = 100% 骗子

## 6. Signature（签名）

**一句话**：用私钥对交易"盖章"——证明操作是你发起的，内容没有被篡改。

**例子**：Sepolia 转账时点"确认"就是签名，像签了字的合同，不可否认或修改。

**⚠️ 安全提醒**：签名不可逆，签前一定要看清内容。恶意 DApp 可能让你签名"授权"来转走资产。

## 7. Transaction（交易）

**一句话**：在区块链上的一次"操作记录"——转账、调用合约、部署合约，都是一笔交易。

**例子**：Sepolia 上转 0.01 ETH 就是一笔交易，记录在链上，任何人都能在 Etherscan 查到。

**误区**：交易不一定是"转账"，调用合约方法（如 Uniswap 兑换）也是交易。查看链上数据不需要交易——那是免费的"读"操作。

## 8. Gas（燃料费）

**一句话**：在区块链上执行操作的"手续费"——就像寄快递要付邮费。

**例子**：Sepolia 转账 Gas 费几乎为零（测试网）。主网简单转账可能几美元，复杂合约调用可能几十上百美元。

**为什么需要 Gas**：给验证者的报酬 + 防止有人滥用网络。测试网 Gas 用的是免费测试 ETH，没有实际价值。

## 9. Smart Contract（智能合约）

**一句话**：部署在区块链上的"自动执行程序"——满足条件就执行，没有中间人，无法篡改。

**例子**：自动售货机类比——投币（满足条件），自动出饮料（执行结果）。Uniswap 合约：存入 Token A，自动按汇率给 Token B。

**⚠️ 安全提醒**：合约一旦部署通常无法修改。调用前要确认是可信的（审计过、开源、社区认可）。

## 10. Testnet（测试网）

**一句话**：区块链的"沙盒环境"——用没有价值的测试币模拟真实操作，专门用来学习和测试。

**例子**：Sepolia 是以太坊测试网，操作流程和主网完全一样，但用免费测试 ETH，没有财务风险。就像飞行模拟器——操作一样，坠机了也没损失。

**误区**：测试网 ETH 没有价值，不能卖钱，不能转到主网。不要花钱买测试网 ETH（有人会骗你）。

## 11. Block Explorer（区块浏览器）

**一句话**：区块链的"搜索引擎+账本查询"——查任何地址余额、任何交易详情、任何合约代码。

**例子**：Etherscan 可以查交易哈希、发送方、接收方、金额、Gas 费、区块号。这是 Web3 的"透明账本"。

## 🔐 为什么私钥、助记词、签名和授权需要特别谨慎

| 概念 | 风险 | 为什么危险 |
|------|------|-----------|
| **私钥** | 🔴 极高 | 拥有私钥 = 拥有全部资产控制权 |
| **助记词** | 🔴 极高 | 能推导出所有私钥 |
| **签名** | 🟠 高 | 不可逆，签了恶意交易 = 资产被转走 |
| **授权** | 🟠 高 | 合约可随时转走你的资产 |

**核心原则**：
1. 私钥和助记词：永远不上网、永远不给别人、永远不输入任何网站
2. 签名前：看清内容，不明白就不签
3. 授权前：确认合约地址是官方的，授权金额尽量小
4. 验证：所有操作做完后去区块浏览器验证结果

Web3 里没有"客服帮你找回资产"这回事。你的私钥、你的责任。

## 概念关系图

```
Address（链上身份）
  ← Wallet（管理工具）
    ← Private Key（控制钥匙）
      ← Seed Phrase（终极备份）

Transaction（链上操作）
  ← Signature（确认操作）
    ← Gas（手续费）
      ← Smart Contract（操作目标）

Testnet（安全练习场）
  ← Block Explorer（查看窗口）
```
