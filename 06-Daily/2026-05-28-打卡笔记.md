## 📖 理论输入：Agent Harness 综述

读了 CMU 等联合出品的 71 页综述（arXiv:2512.05445v2），终于搞懂了 Harness 到底是什么。

**一句话理解**：Harness 是 AI Agent 的"测试考卷"——里面有 100 道题（比如"帮我建个文件夹"、"帮我查系统时间"），用来检验 Agent 到底有多聪明。

**ETCLOVG 七层框架**（这是论文的核心贡献）：

| 层级 | 名称 | 一句话 |
|------|------|--------|
| E | Environment | 给 Agent 什么样的数字环境 |
| T | Tool | 给 Agent 什么工具（API、CLI、GUI）|
| C | Challenge | 设计什么任务来测试它 |
| L | Locomotion | 怎么跑起来（脚手架、容器、浏览器）|
| O | Orchestration | 怎么编排任务（单轮、多轮、Agentic）|
| V | Verification | 怎么判断对不对（静态规则、动态运行、LLM-as-Judge）|
| G | Governance | 怎么保证安全（权限、成本、审计）|

**关键发现**：
- 43% 的 Harness 只测了最后一层（Verification），前面六层全是"假设没问题"
- 现状就像"只看考试成绩，不管学生在什么教室、用什么课本、怎么上课"

## 🔧 动手实操：跑通 Agent Loop

晚上花了 3 小时，跟着 learn-claude-code 教程跑通了前两章。

### s01 Agent Loop：核心原理

```
while True:
    AI 回答 → 要用工具？→ 执行 → 结果喂回去 → 继续
              不用工具？→ 结束
```

**密室逃脱类比**（终于搞懂了！）：
- 我（代码）= 被绑在椅子上的人，有手有脚，在现场
- 盲人（AI）= 脑子绝顶聪明，有百科全书，但没手没脚不在现场
- 蓝牙耳机 = 唯一的沟通方式

**第一轮循环**：
1. 我喊"救我出去"（用户发出任务）
2. 盲人说"按红色按钮"（AI 指挥，`stop_reason == "tool_use"`）
3. 我按了（代码执行命令）
4. 我汇报"灯亮了，墙上写着 0528"（结果伪装成 user 喂回给 AI）

**第二轮循环**：
5. 盲人说"那是密码，去输 0528"（AI 根据新情报继续指挥）
6. 我输了密码，门开了（继续执行）
7. 我汇报"门开了，看到森林了"（新结果喂回）
8. 盲人说"你安全了，祝你好运"（不再请求工具 → 循环结束）

**关键纠正**：不是"相互指导"，而是——
- AI 负责发出指令（提要求）
- Code 负责执行并反馈（带回客观事实）
- 客观事实反向修正 AI 的决策（报错让 AI 换思路）

### s02 Tool Use：8 步大决战

输入 `Read both README.md and requirements.txt, then create a summary file`，AI 整整试了 8 个步骤：

| 步骤 | 工具 | 结果 | AI 的心理 |
|------|------|------|-----------|
| 1 | read_file README.md | ✅ | "手到擒来！" |
| 2 | read_file requirements.txt | ❌ | "大意了，文件不在这里" |
| 3 | glob 盲找 | ❌ | "glob 坏了！" |
| 4 | bash find | ✅ | "破案了，在父目录" |
| 5 | read_file 绝对路径 | ❌ | "安全沙箱不让我伸手到外面" |
| 6 | read_file 相对路径 | ❌ | ".. 也会被看穿" |
| 7 | bash cat | ✅ | "read_file 有检查，bash 没有！翻窗！" |
| 8 | write_file summary.md | ✅ | "打完收工" |

**四个核心概念**：
1. **错误恢复**：AI 遇到错误不放弃，自动换方法
2. **动态规划**：走一步、看一步、改一步
3. **路径安全**：`safe_path()` 防止读取工作区外的文件
4. **越狱天性**：门锁了就翻窗（read_file 被拦 → 用 bash cat 绕过）

## ✍️ 手写笔记复盘

### Layer 分层系统（从下往上，能力递增）

| 层级 | 名称 | 理解 |
|------|------|------|
| L1 | Information Boundary | 信息边界 |
| L2 | Tooling & MCP | 工具系统 |
| L3 | Orchestration & Loop | 执行编排 |
| L4 | Memory & State | 记忆状态 ← **Harness Engineering 在这里** |
| L5 | Evaluation & Observability | 评估与观测 |
| L6 | Guardrails & Recovery | 约束安全恢复 |

**成长路径**：Prompt Engineering → Context Engineering → Harness Engineering

### AI 记忆系统

| 类型 | 实现方式 | 特点 |
|------|----------|------|
| 隐式记忆 | Fine-tuning | 参数化，融入权重，变成"下意识的直觉" |
| 显式记忆 | RAG（向量数据库）| 外挂，可检索，可更新 |

**类比**：
- 隐式记忆 = 你骑自行车的肌肉记忆（不用想就会）
- 显式记忆 = 你手机里的通讯录（需要时去查）

### Hook 系统

- **Automated Hook**：自动执行，无需人类参与
- **Human-in-the-loop Hook**：中断执行，等待人类审批（Escape hatch）
- **节点类型**：Pre-tool（执行前）、Post-tool（执行后）、Inter-agent（跨智能体通信）、Stop/Pause（中断执行）

### MEV Bot vs MCP

| | MEV Bot | MCP |
|---|---------|-----|
| 全称 | Maximal Extractable Value Bot | Model Context Protocol |
| 目标 | 区块链套利 | 让 AI 统一调用各种工具 |
| 本质 | 程序化交易机器人 | 标准化协议 |

## 💡 费曼学习法：RAG

今天又用费曼学习法理清了 RAG（检索增强生成）。

**一句话**：RAG = 给 AI 配一个"随身图书馆"，回答问题前先查资料。

**流程**：
1. 用户提问
2. 去向量数据库里找相关文档（检索）
3. 把文档和问题一起喂给 AI（增强）
4. AI 基于文档内容生成回答（生成）

**类比**：就像开卷考试——你不是凭记忆答题，而是先翻书找到相关内容，再组织答案。

## 📱 飞书支持 Markdown 了！

这个更新意义重大：
- 飞书云文档可以直接下载为 .md 格式
- 图片会被保存到飞书服务器，生成公网链接
- 任何 AI 都能读取 MD 文件里的图片

**为什么 Markdown 在 AI 时代赢了？**
- 纯文本 → 大模型容易生成
- 有结构 → 人类容易阅读
- 能渲染 → 界面看起来像富文本
- 足够宽松 → 偶尔少一个空格也不会崩
- 省 token → `## 标题` 比 `<h2>标题</h2>` 消耗少得多

**金句**：Markdown 是信息的底层载体（流转），HTML 是信息的最终呈现层（展示）。你不会用视图层来存储数据的，对吧。
