1."隐私不是开关，而是基础能力"。这句话让我想到昨天学的 Agent 安全——安全也不是开关，而是架构设计。**两个领域的底层逻辑是相通的：都是在"开放"和"保护"之间找平衡**。🤔

2.老派赛博朋克是为专家准备的，要求全栈人才；新派赛博朋克则说"你可以从一个工具开始"。这让我想到自己——我也是零基础开始的。*技术的门槛不应该是阻止人们参与的理由*。😄

3.*当 Agent 开始动用资金的那一刻，信任就不再是一个应用层的事情，而必须提升到基础层面*。应用层的信任（Prompt 约束、if-else）在 Agent 管钱时彻底失效，必须用智能合约、TEE 硬件安全、零知识证明来锁死底线。「补充昨天」

4.隐私是成千上万个微小的有意识的选择，而安全是成千上万个微小的架构决策。

5.一个有趣的想法：Neo-Cypherpunk 的 CROPS 原则（抗审查、开源、隐私、安全、无需许可）不只是 Web3 的价值观，其实也是 **Agent 安全的基础设施标准**。如果一个 Agent 系统能满足这五条，是不是可以认为它就值得信任。🤔

9.关于 HTML vs Markdown 的思考：Markdown 适合人读，HTML 适合 AI 读。在给 AI 写"项目记忆手册"时，HTML 可以包含更丰富的结构和元数据，让 AI 理解得更准确。工具的选择应该根据"谁来用"来决定，而不是根据"谁来写"来决定。【思考有点片面👀】

10.读了 Claude Code Agent Teams 的文章，发现 **Subagents 是实习生各自给领导交作业，Agent Teams 是小组同事坐一起能互相讨论**。多智能体协作的核心不是"并行"，而是"协调"——共享任务列表、Hooks 质量门、计划审批，都是在建立协调机制。🤝

11.本地部署的 Agent 产品普通人玩不转，市场真正缺的是不用折腾、注册就能用的云端 Agent 团队。从「我派活它干活」→「它提醒我该把这活儿固定下来」，这个转变才是 Agent 产品的核心价值。一个人，确实能活成一支队伍了。💪

12.读了 Datawhale 的 Agent Harness 最小版，理解了 **Agent = model + harness** 的完整含义。Harness 不只是"安全护栏"，而是把 Agentic model 放进一个可运行、可记录、可评分的小环境里。评测的价值不只是"试一下好不好用"，而是能分析问题出在任务理解、工具选择、参数填写、结果读取、步骤冗余，还是评分规则本身不清楚。🎯
