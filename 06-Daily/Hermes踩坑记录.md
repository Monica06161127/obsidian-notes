## 正文

### 开头

一个金融专业大一小白，零编程基础，想在Telegram上配一个24小时在线的AI学习助手。本以为照着教程走就行，结果每一步都有新惊喜😅

记录一下我踩过的坑，给同样想折腾的同学避避雷。

---

### 踩坑实录

#### 🔹 坑1：复制命令也能出错

教程里写的是：
```
hermes gateway setup # 适用于所有消息平台的交互式设置
```
我直接全选复制粘贴，结果报错。排查半天才明白——`#` 后面的中文是**给人看的备注**，不是给电脑的命令！终端看到 `#` 就认为后面全是注释，但有些情况下会把中文当成参数。

💡 **教训：复制终端命令时，`#` 及后面的中文要删掉，只留命令本身**

---
### 踩坑实录
#### 🔹 坑1：开了梯子，终端还是裸奔

配好之后给Telegram机器人发消息，死活没反应。在终端跑 `curl https://api.telegram.org`，直接超时。

原因：macOS 的终端**不会自动走系统代理**。你浏览器能上外网 ≠ 终端也能。

解决：手动给终端设代理，还写了个快捷命令放进 `~/.zshrc`，以后终端输入 `proxy` 就能一键开启：
```bash
alias proxy='export https_proxy=http://127.0.0.1:7890;export http_proxy=http://127.0.0.1:7890'
```

💡 **教训：终端和浏览器是两套网络，开了VPN终端可能还在裸奔**

---

#### 🔹 坑2：API Key 里混进了"看不见的脏东西"

选好模型提供商后，填入 API Key，结果每次都报 HTTP 400。

查了半天才发现：终端输入 Key 的时候，按了方向键，Key 里被塞进了**看不见的控制字符**（类似 `\u001b[B` 这种）。肉眼看不出来，但服务器认不出来就直接拒绝了。

解决：别手打，复制粘贴 Key，粘贴后先看看有没有奇怪字符。

💡 **教训：API Key 这种东西，复制粘贴最安全，手打容易被"暗伤"**

---

#### 🔹 坑3：程序已经在跑了还要再启动

报错说 `Gateway already running (PID 6467)`，我一开始以为出错了。

其实就是：**程序已经在后台运行了，不能再开一个**。终端其实在说"我已经在干活了，别重复启动"。

💡 **教训：遇到 "already running" 别慌，用 `restart` 重启就行**

---

#### 🔹 坑4：机器人收到消息了，但就是不回复

配好 Telegram bot 后，发消息过去石沉大海。逐项排查：

- ✅ 网络通了（代理设好了）
- ✅ Gateway 在运行
- ✅ Bot token 填对了
- ❌ **User ID 没配对！** 机器人有安全机制，只回复"白名单"里的用户，ID 不对就直接忽略，连个"你没权限"的提示都没有。

💡 **教训：机器人"已读不回"很可能是用户ID没配对，用 @userinfobot 仔细核对**

---

### 成果

虽然踩了一堆坑，但最终还是配好了！现在我在Telegram上有一个24小时在线的AI学习助手，帮我看文档、管笔记、做打卡记录。

最大的感受是：**终端报错看着吓人，但大部分都是"说人话"的，只是它说的不是中文😂** 耐心读一读，总能读懂。

---

### 给同样想折腾的同学

1. 终端≠浏览器，开了梯子终端可能还是裸奔
2. 复制命令时记得删掉 `#` 后面的中文
3. API Key 别手打，直接粘贴
4. 遇到 "already running" 用 restart
5. 机器人不回复，先查用户ID
