## 一、今天做了什么

### 1. LXDAO 讲座：Neo-Cypherpunk & Privacy

参加了 LXDAO 的线上讲座，嘉宾是 **Peter Farbey**（W3PN / Web3 Privacy Now）。

**讲座主题**：从 Neo-Cypherpunk 与隐私文化的角度，讨论为什么隐私不是少数人的附加选项，而是 AI × Web3 Builder 需要理解的基础能力。

**核心内容**：

#### W3PN 组织介绍
- 非营利性智库，过去三年举办了大量活动
- 超过 15000 人参加，来自 70 个国家
- 200 位演讲者，从切尔西·曼宁到杰克·多西，从唐凤到 Vitalik Buterin
- 在活动家和自由科技之间建立团结

#### Neo-Cypherpunk 概念
- 起源于 1993 年的《赛博朋克宣言》
- 最初是一群黑客希望**密码学公开、可访问、开源**
- "Neo"= 新，是**隐私 + 游戏 + 关怀 + 社区**的混合体
- 不再是专家专属，**任何人都能参与**

#### 老派 vs 新派赛博朋克
- **老派**：为专家准备，要求全栈人才，个人主义
- **新派**：所有水平都能参与，可以从一个工具开始，**集体主义**

#### CROPS 原则
- **C**ensorship-resistant（抗审查）
- **O**pen-source（开源）
- **P**rivacy（隐私）
- **S**ecurity（安全）
- **P**ermissionless（无需许可）

#### 关键观点
- **隐私是实践，不是理论**："成千上万个微小的有意识的选择"
- **没有标准答案**：每个人的需求不同（举报人 vs 普通用户）
- **多元主义**：不拘泥于一种工具，使用适合自己的
- **从个人到集体**：相互保护，隐私赋能社区

#### 实用资源
- **隐私构建器**（免费开源）：词汇表、工具、创意生成器、案例、框架
- **Essentials Explorer**：800+ 隐私项目，可查看隐私级别
- **个人隐私文本 #personalprivacy**：Vitalik 等人分享的隐私工具
- 下一场活动：**6 月 14 日，柏林**，新丝绸之路峰会（免费）

> 🤔 感想：Peter 说"隐私不是开关，而是基础能力"，这句话让我想到昨天学的 Agent 安全——安全也不是开关，而是架构设计。两个领域的底层逻辑是相通的。

### 2. Gemini 对话：Agent 记忆与安全探索

今天和 Gemini 深入讨论了几个 AI × Web3 的核心话题：

#### 2.1 Coding Agent 的记忆系统
- **现状**：用 `.cursorrules` / `memory-bank.md` 给 AI 写"项目记忆手册"
- **进化**：AI 已经能**自己更新记忆**（踩坑后主动记录）
- **未来**：隐式记忆（向量数据库 + 知识图谱，自动从 Git 历史提取）

#### 2.2 Agent 资金安全
- **核心洞察**：当 Agent 开始动用资金，信任必须从应用层下沉到基础设施层
- **应用层信任**（Prompt 约束、if-else）在 Agent 管钱时**彻底失效**
- **基础设施信任**：智能合约、TEE 硬件安全、零知识证明
- **核心架构**：意图与执行分离——AI 负责"动脑"，安全沙盒负责"动钱"

#### 2.3 安全 vs 能力的跷跷板
- **能力 = 开放度**（给更多记忆、更多权限）
- **安全 = 隔离度**（最小权限、严格限制）
- 两边不能极端，需要找**黄金支点**

#### 2.4 滑点（Slippage）概念
- 在 DEX 交易中，报价和成交价之间的差异
- 用拍卖类比：你出价 100 元，但成交时涨到 102 元
- 高滑点 = 价格变动大，可能被 MEV 机器人攻击

#### 2.5 HTML vs Markdown：为什么现在 HTML 更好用
- **Markdown 的优势**：简洁、易读、适合写文档
- **HTML 的优势**：更灵活、更强大、适合现代 Web 开发
- **关键洞察**：在 AI 时代，HTML 的表达能力更强，可以嵌入交互组件、样式、脚本
- **实际应用**：给 AI 写"项目记忆手册"时，HTML 可以包含更丰富的结构和元数据
- **未来趋势**：随着 AI 工具的发展，HTML 可能成为更主流的"AI 可读"格式

### 3. 收到专业评论认可（Harness Engineering）

在小红书打卡笔记下收到一条专业评论，说我的思考对应了 **Harness Engineering** 的六大项。

**Harness Engineering** 是专门研究"如何给 AI Agent 施加安全缰绳"的工程学科，包含六大支柱：
1. **Intent Alignment**（意图对齐）
2. **Least Privilege**（最小权限）
3. **Policy Compliance**（策略合规）
4. **Audit Trail**（审计追踪）
5. **Sandbox Isolation**（沙盒隔离）
6. **Human-in-the-Loop**（人类在环）

> 🎉 这条评论说明我的思考已经和专业理论对齐了！写的 Agent 安全观点精准对应了 Harness 的核心框架。

### 4. 阅读：Claude Code Agent Teams（多智能体协作）

读了一篇关于 Claude Code 中 **Agent Teams** 功能的完整工程指南。

**核心概念**：
> **Subagents 是实习生各自给领导交作业，Agent Teams 是小组同事坐一起能互相讨论。**

**Subagents vs Agent Teams**：
- **通信**：Subagents 只向主代理汇报 → Agent Teams 队友之间直接收发消息
- **协调**：Subagents 主代理统一管理 → Agent Teams 共享任务列表 + 自协调
- **Token 成本**：Subagents 较低 → Agent Teams 较高（每个队友是独立实例）

**什么时候开团**：
- ✅ 研究和审查类、新模块开发、Bug 调试、跨层协调
- ❌ 顺序依赖强、同一文件多人编辑、依赖关系多

**最佳实践**：
- 团队规模 3-5 个最合适
- 给队友足够的 context（不会继承 lead 的对话历史）
- 任务粒度要恰到好处
- 从研究和审查类任务入手
- 用 Hooks 强制质量门（TeammateIdle、TaskCreated、TaskCompleted）

### 5. 阅读：天工超级智能体（Skywork 国内版）

读了一篇关于**云端 Agent 产品**的体验文章。

**核心观点**：
> 本地部署的 Agent 产品（OpenClaw、Hermes Agent）普通人玩不转，市场真正缺的是**不用折腾、注册就能用的云端 Agent 团队**。

**三个实测场景**：
1. **我睡觉，它盯盘**：连接飞书，定时任务自动盯美股，智能处理节假日顺延、数据源被拦自动换源
2. **顺着兴趣，把赛道讲透**：一句话生成 11 页 PPT（AI 算力生意），配色克制、数据挂来源
3. **替我把不熟的东西调研明白**：深度调研 Codex vs Claude Code，后台任务出错自动重启、换方式重跑

**自研模型 SkyClaw-v1.0**：
- Agent 评测基准第一梯队
- 价格是同级模型的一半甚至更低
- 注册免费试用

**核心转变**：
> 从「我派活它干活」→「它提醒我该把这活儿固定下来」。一个人，确实能活成一支队伍了。

### 6. 阅读：Agent Harness 最小版（Datawhale）

读了 Datawhale 陈思州整理的 **Agent 评测框架**。

**核心观点**：
> **Agent = model + harness**
> Harness 是把 Agentic model 放进一个可运行、可记录、可评分的小环境里。

**5 个核心模块**：
- **Task**：任务输入
- **Environment**：可操作环境（代码仓库、文件集）
- **Tools**：工具接口（read_file、list_files、run_tests）
- **Trace**：执行记录（每一步用了什么工具、参数、返回）
- **Grader**：评分器（规则、测试脚本）

**Harness 的价值**：
> 不只是"试一下 Agent 好不好用"，而是能分析问题出在**任务理解、工具选择、参数填写、结果读取、步骤冗余，还是评分规则本身不清楚**。

## 二、核心收获

### 今天的思维主线

**一句话总结**：
> Agent 越自主（记忆越深、能力越强、能动资金），就越需要**边界**来保护它——安全和能力永远是跷跷板的两端。

### 三个核心洞察

**1. 隐私是基础能力，不是附加选项**
- Peter 说"隐私不是开关，而是基础能力"
- 结合昨天学的 Agent 安全：安全也不是开关，而是架构设计
- **Neo-Cypherpunk 的 CROPS 原则** = **Agent 安全的基础设施标准**

**2. Agent 资金安全必须下沉到基础设施层**
- 应用层的信任在 Agent 管钱时失效
- 必须用智能合约、TEE、零知识证明锁死底线
- **意图与执行分离**是核心架构原则

**3. 安全与能力的跷跷板**
- 能力 = 开放度（记忆、权限）
- 安全 = 隔离度（最小权限、严格限制）
- 需要找**黄金支点**，不能极端

### 今天的完整知识拼图

今天最大的收获是发现**六个领域的底层逻辑是相通的**：

| 维度 | 内容 | 来源 |
|------|------|------|
| **隐私与安全** | Neo-Cypherpunk、CROPS 原则 | Peter 讲座 |
| **Agent 记忆** | Memory Bank、自动更新、隐式记忆 | Gemini 对话 |
| **Agent 资金安全** | 意图执行分离、基础设施信任 | Gemini 对话 |
| **Agent 协作** | Agent Teams、共享任务列表、Hooks 质量门 | Claude Code 文章 |
| **Agent 产品化** | 云端团队、定时任务、自动排障 | 天工超级智能体 |
| **Agent 评测** | Harness 五模块、Eval Case、Trace 记录 | Datawhale |

**一句话串联**：
> Agent 越自主（记忆越深、能力越强、能动资金），就越需要边界来保护它——而 **Harness 就是这个边界的工程化实现**，从评测（Trace + Grader）到协作（Hooks + 共享任务列表）到产品化（自动排障 + 定时任务），都是同一个逻辑。

### 跨领域串联

今天最大的收获是发现**三个领域的底层逻辑是相通的**：

| 领域 | 核心问题 | 解决思路 |
|------|---------|---------|
| AI 记忆 | 你的数据归谁？ | Memory Bank + 自动更新 |
| 链上隐私 | 你的交易信息归谁？ | CROPS 原则 + 隐私工具 |
| Agent 钱包 | 你的资产归谁？ | 意图执行分离 + 基础设施信任 |

**时间在变，技术在变，但底层问题或许一直没变。**
