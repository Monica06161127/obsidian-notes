## 一、今天做了什么

### 1. Harness 实操：s01 Agent Loop + s02 Tool Use

今天终于把昨天学的理论变成了实操！跟着 learn-claude-code 教程跑通了前两章。

**s01 Agent Loop 实战**：
- 输入：`Create a file called hello.py that prints "Hello, World!"`
- AI 自动创建文件、运行，遇到 `python` 命令失败后自动切换到 `python3`
- **关键观察**：AI 有"错误恢复能力"，遇到问题会自动换方法

**s02 Tool Use 实战**：
- 测试了 4 个任务：读文件、创建+读取、找文件、复杂任务
- 最精彩的是"8 步大决战"：AI 尝试了 8 种方法，失败 4 次，但每次都自动恢复
- **关键发现**：当专用工具（read_file）被安全机制拦住时，AI 会用 bash 绕过——这就是"越狱天性"！

> 🤔 感想：昨天学的"密室逃脱类比"今天在代码里完全验证了！AI 是盲人（有脑子没手），代码是被绑的人（有手没脚），蓝牙耳机是唯一的沟通方式。

### 3. AI × Web3 问题地图（Week 2 Module A 作业）

完成了共学营的 Week 2 作业：画一张 AI × Web3 问题地图。

**核心思路**：
- AI = **生产力**（认知能力）：处理非结构化信息、推理、动态决策
- Web3 = **生产关系**（信任与安全边界）：价值流转、所有权确权、权限隔离
- 交界处的痛点：AI 有幻觉，Web3 门槛高 → 两者互补

**6 个方向拆解**：
1. Payment / Commerce / Settlement
2. Identity / Reputation / Capability
3. Wallet / Permission / Safe Execution
4. Privacy / Security / Sovereignty
5. Dev Tooling / Agent Workflow
6. Governance / Coordination / Public Goods

**深度剖析（2 个方向）**：
- Wallet / Permission：AI 无法保管私钥，Web3 无法理解意图
- Payment / Commerce：AI 无法解决零信任交付博弈，Web3 无法进行主观质量评估

**主线选择**：🔐 Wallet / Permission / Safe Execution
- 最核心的瓶颈："既要自主，又要安全"
- 与 Harness Engineering 直接相关（s02/s03）
- 与金融专业背景契合

> 🤔 感想：这个问题地图让我把昨天学的 Harness Engineering 和今天学的 AI × Web3 串起来了。Agent Loop 是"大脑"，Tool Use 是"双手"，而 Wallet/Permission 就是"神经系统"——控制大脑和双手的边界。

## 二、核心收获

### 今天的思维主线

**一句话总结**：
> 从"理解 Agent Loop 原理"到"画出 AI × Web3 问题地图"，我终于看清了 Harness Engineering 在整个版图中的位置。

### 收获详情

**1. Agent Loop 的本质验证**
- 昨天用密室逃脱类比理解了 Agent Loop
- 今天在代码里亲眼看到：AI 自动修复错误、自动切换工具、自动恢复失败
- **关键认知**：Agent Loop 不是"相互指导"，而是 AI 出脑子、代码出双手、客观事实反向修正 AI

**2. Tool Use 的设计哲学**
- s01 只有 bash（万能但不安全）
- s02 加了专用工具（read_file、write_file 等，安全但有限制）
- **关键认知**：工具越多，AI 越强大，但也越危险——这就是为什么需要 s03 Permission

**3. AI × Web3 的交界线**
- AI 擅长"不确定"：推理、决策、处理模糊信息
- Web3 擅长"确定"：转账、确权、权限隔离
- **关键认知**：两者结合，就是用 Web3 的确定性规则去规范 AI 的不确定行为

**4. Wallet/Permission 是最核心的瓶颈**
- AI 有"越狱天性"：门锁了就翻窗（read_file 被拦 → 用 bash 绕过）
- Web3 有"硬性边界"：私钥、权限、智能合约
- **关键认知**：解决"既要自主，又要安全"的矛盾，是整个赛道的关键
