# 2026-05-18 学习记录 — Week 1：AI Native × Web3 Native + 开发者工具链

## 今天学了什么

### 🧠 核心概念：AI × Web3 范式

**AI Native（AI 原生）**
- 定义：从设计之初就把 AI（大模型）作为核心底层组件，去掉 AI 就无法运行
- 对比：AI 赋能 = 传统产品 + AI 插件（去掉 AI 还能跑）；AI Native = AI 是心脏（去掉就死）
- 特征：模型底层嵌入、自然交互主导、数据闭环迭代、不可移除性
- 例子：ChatGPT、Perplexity、Cursor

**Web3 Native（Web3 原生）**
- 定义：从设计之初就把区块链、去中心化、Token 经济作为 DNA
- 对比：Web2.5 = 传统产品 + 区块链插件（去掉还能跑）；Web3 Native = 区块链是 DNA（去掉就不是它了）
- 六大特征：去中心化架构、用户真实所有权（Read+Write+Own）、Token 原生经济、信任最小化、可组合性（货币乐高）、无需许可
- 例子：Uniswap、Aave、Decentraland

**关键洞察：AI Native × Web3 Native 正在融合**
- AI Agent + 链上执行 = 新趋势
- 两个范式都强调"不可移除性"

### 🔌 协议层：Agent 如何连接世界

**MCP（Model Context Protocol）**
- 连接大模型与外部数据源/工具的统一标准协议
- 让 AI 能"看到"外部世界

**ACP（Agent Client Protocol）**
- 规范代码编辑器与 AI 编码助手之间的通信协议
- 让 AI 能"操作"代码编辑器

**Function Call**
- LLM 调用外部函数/工具的底层机制
- MCP/ACP 建立在 Function Call 之上

### 🛠️ 开发者工具链（GitHub 生态）

**核心工作流：Issue → PR → Release**
- Issue：问题记录本 + 任务看板
- Pull Request：代码变更的正式合并申请
- Release：正式版本发布
- git push：本地代码上传到 GitHub

**GitHub CLI（gh）**
- 终端里操作 GitHub 的命令行工具
- 配置要点：需要设置代理（export https_proxy）才能连接外部服务

**HTTPS vs SSH**
- HTTPS：用账号密码/Token 认证，简单但需输入凭据
- SSH：用密钥对认证，配置一次后免密

**其他工具**
- Gist：轻量级代码片段分享
- Codespaces：云端完整 VS Code 开发环境

### 💻 编程基础

**Typeless（无类型语言）**
- 没有类型系统的语言（如 BCPL、汇编）

## 核心收获

1. **"不可移除性"是判断 Native 的金标准**：去掉这个技术，产品还能活吗？能活 → Enabled；不能活 → Native
2. **MCP + ACP = AI Agent 的"五官和四肢"**：MCP = 感知；ACP = 操作；Function Call = 神经系统
3. **GitHub 工具链 = 现代开发者的"操作系统"**：Issue → PR → Release = 完整闭环
