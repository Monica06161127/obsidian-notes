## 踩的坑（重点！）

### 坑 1：pip 命令找不到
**现象**：`zsh: command not found: pip`
**原因**：macOS 上 Python 3 用 `pip3`，不是 `pip`
**解决**：用 `pip3 install` 或 `python3 -m pip install`

### 坑 2：requirements.txt 找不到
**现象**：`No such file or directory: 'requirements.txt'`
**原因**：在错误的目录执行了 git clone，导致目录结构混乱
**正确操作**：
```bash
cd ~  # 先回到 home 目录
git clone https://github.com/shareAI-lab/learn-claude-code.git
cd learn-claude-code  # 进入仓库根目录
```

### 坑 3：API Key 被中文引号包围
**现象**：`UnicodeEncodeError: 'ascii' codec can't encode character '\u201c'`
**原因**：从网页复制 API Key 时，带了中文引号 `""`
**教训**：粘贴 API Key 时，检查是否有多余的引号或空格

### 坑 4：API Key 无效
**现象**：`Invalid API Key`
**原因**：
- 一开始用了过期的 key
- 后来用了错误的 key（sk- 开头的是另一个服务的）
- tp- 开头的 key 需要用正确的 base_url
**解决**：
- 确认 key 类型（tp- 开头用 MiMo 的 base_url）
- 确认 base_url（中国区用 `token-plan-cn`，新加坡用 `token-plan-sgp`）

### 坑 5：Base URL 配置
**现象**：s02 一开始报 401 认证错误
**原因**：s01 硬编码了 base_url，但 s02 用环境变量读取
**解决**：在 `.env` 文件中添加 `ANTHROPIC_BASE_URL=https://token-plan-cn.xiaomimimo.com/anthropic`

### 坑 6：Python 版本不兼容
**现象**：`TypeError: unsupported operand type(s) for |: 'type' and 'NoneType'`
**原因**：代码用了 Python 3.10+ 的语法 `int | None`，但系统是 Python 3.9
**解决**：改成字符串注解 `"int | None"` 或用 `Optional[int]`

### 坑 7：glob 工具有 bug
**现象**：`glob() got an unexpected keyword argument 'root_dir'`
**原因**：s02 的 glob 工具代码有 bug
**表现**：AI 遇到错误后自动切换到 bash 工具完成任务
**学习点**：这反而展示了 AI 的错误恢复能力！

## 核心收获

### 1. Agent Loop 的本质
```
while True:
    AI 回答 → 要用工具？→ 执行 → 结果喂回去 → 继续
              不用工具？→ 结束
```
就像雇了一个助手：你说任务 → 助手决定要不要动手 → 动手就执行 → 结果告诉你 → 继续或结束

### 2. 工具分发机制
```python
TOOL_HANDLERS = {
    "bash":       run_bash,
    "read_file":  run_read,
    "write_file": run_write,
    ...
}
# AI 说用什么工具 → 去字典里找对应的函数 → 执行
```
就像工具架：每种工具放在固定位置，用的时候直接拿

### 3. AI 的错误恢复能力
- 遇到错误不放弃，自动尝试其他方法
- 能规划多步骤任务
- 能根据结果动态调整策略

### 4. 真实世界的复杂性
- API Key 可能过期、被引号包围、格式不对
- 文件可能不在预期位置
- 工具可能有 bug
- 安全限制会阻止某些操作
