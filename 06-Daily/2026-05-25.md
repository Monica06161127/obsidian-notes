## 一、今天做了什么

### 1. 听讲座：AI Agent 的 Memory 问题

今天共学营的讲座主题是「Long-term Memory for AI Agents」，这是 Week 2 的核心话题。

**核心问题**：为什么 ChatGPT 每次对话都"失忆"？

讲座从三个层面展开：

**（1）为什么需要 Memory？**
- 现在的 AI 每次新对话都"重置"——用户偏好、项目上下文、决策历史全部清零
- 目标是让 AI 从"一次性工具"变成"长期伙伴"
- 核心转变：ChatGPT 正在从"工具"变成"伙伴"，前提就是 Memory

**（2）Memory 的技术挑战**
- 记忆会"打架"：前后矛盾的偏好怎么调和？
- 检索比存储更难：存了一屋子书，但找不到你需要的那段话
- 所有权问题：你的 AI 记忆属于你还是平台？（涉及 GDPR）

**（3）编程 Agent 是最需要 Memory 的场景**
- 持续几周的软件项目，AI 需要记住技术栈、决策历史、失败方案
- 现在的解决方案：用 .md 文件（如 AGENTS.md）给 AI 写"项目记忆手册"
- 未来方向：AI 自己维护记忆，Session 重启后恢复工作

**（4）记忆工程（Memory Engineering）**
- 现在的 AI 记忆系统还很"缝合怪"——数据库+向量检索+上下文管理拼凑
- 未来可能成为像"Prompt Engineering"一样的重要方向
- Benchmark 指标：Recall（召回率）、Retrieve（检索）、Revise（修正）

详细笔记 → [[02-AI基础/Agent记忆系统]]

### 2. 共学营讨论：Web3 隐私话题

下午参加了共学营的 co-learning 环节，讨论了链上隐私。

**核心发现**：
- 区块链不是"匿名"的，是"透明"的——所有交易公开可查
- RPC 节点会记录你的 IP 地址，很多 RPC 架设在 AWS 上——去中心化的承诺败给了中心化基础设施
- 🥔 老师说得很形象："在 Web3 里面，大家其实是裸奔的"

**隐私协议对比**：
- **Tornado Cash**：支票模型，存进去后钱就"冻结"了，不能边隐私边交易
- **Railgun**：UTXO 模型（像独立钞票），可以在隐私状态下去 DeFi 里操作

**关键设计**：
- Viewing Key 和 Spending Key 分离：给审计机构看"清白证明"，但他们动不了你的钱
- POI（来源完整性证明）：资金进隐私池后自动检查来源是否干净
- 反直觉结论：越多人用隐私协议，隐私性反而越好（"大海捞针"效应）

## 二、核心收获

### 收获 1：AI 的 Memory 问题本质是"信任"问题

你愿意让 AI 记住多少关于你的信息？记住了之后怎么保证安全？这不只是技术问题，更是伦理和法律问题。

作为金融专业的学生，我发现这个问题和金融世界的"资产确权"高度相似——你的数据是你的"资产"，谁有权使用、谁能带走、怎么保护，都需要明确的规则。

### 收获 2：透明 vs 隐私是 Web3 的核心矛盾

区块链的设计初衷是"公开透明"，但这和个人隐私、商业机密产生了冲突。隐私协议（如 Railgun）就是在透明和隐私之间找平衡。

有意思的是，Agent Memory 和链上隐私本质上在回答同一个问题——**在透明的世界里，如何保护个体的边界？**

### 收获 3：金融生的"不可替代性"不是代码能力

跟 Gemini 聊完后更清晰了：金融生在 Web3 领域的价值是"看懂数学+会用数据+能设计 Tokenomics"，而不是跟 CS 学生拼代码。找到自己的生态位比盲目补课更重要。

### 收获 4：蒸馏（Distillation）的概念

在共学营里学到了"蒸馏"这个概念——从复杂模型中提炼出简单但有效的知识。这和 Agent Memory 有异曲同工之妙：
- 蒸馏：从内部出发，提炼知识
- Agent Memory：从外部出发，存储和检索知识

一个是从内部"炼化"，一个是从外部"记忆"。

## 四、关键概念速记

| 概念 | 一句话解释 | 生活类比 |
|------|-----------|---------|
| Agent Memory | 让 AI 记住你的长期记忆系统 | 咖啡店店员记住你的口味 |
| RPC | 钱包和区块链之间的"信使" | 邮局 |
| UTXO | 未花费交易输出，独立"钞票"模型 | 你手上的多张钞票 |
| ZK（零知识证明） | 证明某事为真但不泄露细节 | 证明你成年了但不出示身份证 |
| POI | 资金来源完整性证明 | 银行存款时的"资金来源清白证明" |
| Railgun | 新一代隐私协议，UTXO 模型 | 可以边隐身边交易的"隐身斗篷" |
| Tornado Cash | 早期隐私协议，支票模型 | 存支票取支票，但钱不能动 |
| Viewing Key | 只看不花的密钥，用于合规审计 | 给审计师看账本但不给钥匙 |
| Spending Key | 控制交易的密钥 | 保险柜钥匙 |
| Memory Engineering | AI 记忆系统设计的工程方向 | 可能成为下一个"Prompt Engineering" |

## 五、今天的三个模块之间的联系

今天学了三块看似不同的内容，但它们有一个共同的底层问题：

```
AI Agent Memory → 你的数据归谁？
链上隐私        → 你的交易信息归谁？
职业规划        → 你的过程和成果归谁？
```

**本质上，三个话题都在回答同一个问题：在透明的世界里，如何保护个体的边界？**

- AI 记住了你的偏好和项目数据 → 你能不能带走这些记忆？
- 区块链记录了你的每一笔交易 → 你能不能隐藏这些信息？
- 你在学校和社区里产出的内容 → 你能不能把它变成自己的"资产"？

技术在变，但底层问题一直没变。
