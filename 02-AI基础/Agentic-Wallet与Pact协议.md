# Agentic Wallet 与 Pact 协议

> 当 AI Agent 学会了管钱，谁来保证它不会乱花？

## 一句话定义

**Agentic Wallet** 是专门为 AI Agent 设计的加密钱包，让 Agent 能在安全边界内自主管理链上资产、执行交易。**Pact** 是其中的授权协议，像一份"工作说明书"，框定 Agent 能做什么、不能做什么。

---

## 🪜 Agent 的进化阶梯

```
Chatbot ──────→ Agents        （从工具到自主体）
Copilots ─────→ Autonomous Agents （从助手到独立决策者）
```

**生活类比**：
- Chatbot = 计算器（你说算什么就算什么）
- Copilot = 导航软件（给你建议，但你开车）
- Agent = 自动驾驶（它自己做决策）
- Autonomous Agent = 无人出租车（完全自主，连方向盘都没有）

**核心矛盾**：越自主的 Agent 越有用，但也越危险。链上资金需要私钥管理，Agent 拿到私钥 = 拿到钱包的"钥匙"。

---

## ⚠️ Agent 管钱的五大安全风险

### 1. 影子托管（Shadow Custody）

Agent 暗中控制资金，用户不知道它在做什么。

**类比**：请了个管家，但他偷偷用你的银行卡买了东西，你月底看账单才发现。

### 2. EUA 地址池失控

EUA（External User Agent）地址池：Agent 创建的一堆地址，用户管不了。

**类比**：你给了助理一张副卡，结果他办了十张副卡给他的朋友们用。

### 3. 自然语言的模糊性

自然语言不具备强约束权限——"帮我转点钱"到底是转多少？给谁？

**类比**：你跟朋友说"帮我带杯咖啡"，结果他带了杯 88 块的特调。

### 4. Prompt Injection（提示词注入）

攻击者通过注入恶意指令，让 Agent 做出违背用户意图的操作。

**类比**：有人在你的购物清单上偷偷加了一行"买个 iPhone 16 Pro Max"。

### 5. Zombie Permissions（僵尸权限）

授权后忘记撤回，被攻击者利用。

**类比**：你给前室友的门禁卡没注销，结果他还能进你家。

---

## 🛡️ 解决方案：Pact Authorization Protocol

### 核心理念

> 大量资金 → 人类控制，Agent 执行受 Pact 限制。

**Pact = 一份给 Agent 的"工作说明书"**

### 四个核心组件

```
┌─────────────────────────────────────┐
│           Pact 协议                  │
├─────────────┬───────────────────────┤
│ 1. Intent   │ Agent 要完成什么目标    │
│ 2. Plan     │ 具体怎么做（步骤）      │
│ 3. Policies │ 红线在哪（风控约束）     │
│ 4. Complete │ 怎样算做完（完成条件）   │
└─────────────┴───────────────────────┘
```

**生活类比**：就像你请装修队装修房子——
- Intent：把客厅刷成白色
- Plan：先打磨墙面 → 刷底漆 → 刷两遍面漆
- Policies：不能动承重墙，不能超过预算 5000 块
- Complete：墙面均匀无气泡，验收通过

### 可信执行链路

```
submit（提交请求）→ approve（人类审批）→ 强制执行
```

Agent 不能跳过审批直接执行。就像公司报销流程：你提交 → 财务审核 → 打款。

## 🏗️ Agent 钱包架构

### Owner → 多 Agent 模式

```
Owner（你）
  ├── Trading Agent（交易机器人）
  ├── Risk Agent（风控机器人）
  ├── Ops Agent（运维机器人）
  └── ...（更多 Agent）
```

每个 Agent 有独立的权限范围，互不干扰。

### 子钱包 + Layer 保护

- 主钱包分出**子钱包**给不同 Agent
- 每个子钱包加 **Layer 隔离**，一个被攻破不影响其他
- Agent **只能操作固定地址** ✓

**类比**：你有一张主银行卡，给不同用途办了不同副卡（买菜卡、网购卡、加油卡），每张卡有独立限额。

## 🔧 技术组件

### MPC for AI Agents

**MPC（Multi-Party Computation，多方计算）**：把私钥拆成多个片段，分别保管，签名时需要多方协作。

**类比**：银行金库有两把钥匙，行长和副行长各一把，两个人同时转才能开门。

### Recipe Skill Layer

链上协作的技能层——Agent 的"技能包"，定义它能调用哪些链上能力。

### 知识胶囊（Knowledge Capsule）

针对特定场景的知识包，让 Agent 在限定领域内做出准确决策。

### X402 微支付协议

Agent 之间的小额支付协议。HTTP 402 状态码 = "Payment Required"，X402 让 Agent 可以自动完成微支付来获取服务。

### Gasless 场景

用 USDC/USDT 支付 gas 费，降低 Agent 使用门槛。不再需要持有 ETH 来支付手续费。

## ✍️ 学习笔记

这是 Day 9（2026-05-26）Cobo Agentic Wallet 讲座的详细整理。核心感悟：

**"Agent 越自主，越需要边界。"**

Pact 协议的设计思路很有启发——不是限制 Agent 的能力，而是给它一个明确的"工作说明书"。这和我们平时给人授权是一样的：你不会说"随便帮我处理"，而是会说"帮我做这件事，但不能碰那个"。

这让我联想到昨天学的 Agent 记忆系统：记忆让 Agent 更聪明，但记忆泄露 = 资产泄露。安全和能力永远是跷跷板的两端。
