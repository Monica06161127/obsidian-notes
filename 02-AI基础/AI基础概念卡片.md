# 🧠 AI 基础概念卡片

> 用生活类比帮助自己建立 AI 领域的共同语言

## 1. LLM（大语言模型）

**一句话**：读过海量文本的"超级接话王"——不真正理解，但能猜出最合理的下一句。

**例子**：问 Claude "什么是比特币？"，它基于训练时读过的无数文章拼出回答。就像一个从没炒过股但读了一万本财经书的人。

**误区**：LLM 会"一本正经地胡说八道"（hallucination）。答案是"统计上最可能的"，不是"事实正确的"。关键信息一定要自己验证。

## 2. Prompt（提示词）

**一句话**：给 AI 下的"指令"——说得越清楚，回答越好。

**例子**：去餐厅点菜。说"来点吃的"→ 随便上；说"不辣的番茄鸡蛋盖饭，米饭少一点"→ 精准。Prompt 也一样。

**误区**：Prompt 不只是"问问题"，可以设定角色、限定格式、给出示例、要求分步骤思考。更像是在"编程"对话。

## 3. Context Window（上下文窗口）

**一句话**：AI 的"短期记忆容量"——一次能记住多少内容有上限。

**例子**：跟朋友聊微信，1000 条消息他不可能全记住。Context Window 就是"最近能记住的范围"。

**误区**：AI 不能记住所有对话。Claude 约 200K token（一本中等厚度的书），超出就"忘"了。重要信息要重复提醒或写进文件。

## 4. Workflow（工作流）

**一句话**：把复杂任务拆成多个步骤，按顺序或条件自动执行。

**例子**：银行贷款审批：提交申请 → 系统初审 → 人工复核 → 批准/拒绝 → 通知客户。AI Workflow 也一样。

**误区**：不是让 AI 一口气干完所有事。好的 Workflow 每一步可检查、可修改，像分阶段给实习生任务。

## 5. Agent（智能体）

**一句话**：能自己决定"下一步做什么"的 AI——不只是回答问题，还能主动规划和执行。

**例子**：普通 AI 像客服（问一句答一句）；Agent 像私人助理（说"订明天去上海的高铁"，它自己查车次、比较、选座、提醒带身份证）。

**我在用**：Hermes Agent 帮我查日历、读文件、生成摘要、推 GitHub，不需要一步步指挥。

**误区**：Agent 能力取决于工具和指令。没有好的 Prompt 和 Tool，Agent 像没有手机的助理——很聪明但啥也干不了。

## 6. Tool Use（工具调用）

**一句话**：让 AI 能"动手做事"——不只是说话，还能执行操作。

**例子**：AI 本身只能生成文字。加上 Tool Use，能搜索网页、读写文件、调用 API、发送消息。

**我在用**：Hermes 调用 WCB API 查任务、GitHub CLI 推代码、读写 Obsidian 笔记——都是 Tool Use。

**误区**：AI 调用工具是靠提前"教"的。工具描述写得不好，AI 就会用错或不用。

## 7. AI Coding（AI 辅助编程）

**一句话**：让 AI 帮你写代码——能补全、解释、找 bug，但不能替你思考"要解决什么问题"。

**例子**：像用 Excel 公式自动计算，但你得先想清楚"要算什么"。AI Coding 也一样。

**我在用**：Claude Code 帮我写脚本、调试。有一次解析 API 数据，它写出了完整 Python 脚本，但格式不对时还是我自己发现的。

**误区**：AI 写的代码可能有 bug、不安全、不符合场景。AI Coding 是"副驾驶"，你才是"驾驶员"。

## 8. Human-in-the-Loop（人机协作）

**一句话**：在 AI 工作流程中设置"人工检查点"——关键决策由人做，AI 负责执行和建议。

**例子**：自动驾驶汽车——大部分时候自己开，复杂路况提醒驾驶员接管。

**我在用**：Hermes 生成笔记和打卡内容，每篇我都会自己看、改一改再发布。AI 做初稿，我做终审。

**误区**：加了人工审核不等于万无一失。人也会偷懒、也会信任 AI 而不仔细看。真的认真审核才是 Human-in-the-Loop。

## 概念关系图

```
Prompt → LLM → Context Window → Agent → Tool Use → Workflow → Human-in-the-Loop
                                                                      ↑
                                                              AI Coding 实现这一切
```

像一条生产线：Prompt 是订单，LLM 是工人，Context Window 是工作台大小，Agent 是工头，Tool 是工具箱，Workflow 是生产流程，Human-in-the-Loop 是质检员。
