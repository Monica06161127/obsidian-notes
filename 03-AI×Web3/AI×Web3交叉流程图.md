# 🔄 AI × Web3 最小交叉流程图

> 基于 Week 1 真实经历：用 Hermes Agent 完成测试网交易

## 流程图：测试网交易引导

```
用户                          AI / Agent                    链上操作
────                          ──────────                    ────────

① 发起任务
"我想转 0.01 ETH"
     │
     ▼
                              ② AI 规划
                              • 解释交易含义
                              • 检查地址格式
                              • 预估 Gas 费
                              • 生成交易摘要（草稿）
     │
     ▼
┌─────────────────────────────────────────────────────────────────┐
│  ⚠️ 人工确认点 #1：审查交易摘要                                  │
│  确认：地址正确？金额正确？网络正确？                              │
└─────────────────────────────────────────────────────────────────┘
     │
     ▼
                              ③ AI 提供操作指引
                              "在 MetaMask 中：发送→粘贴地址→确认"
     │
     ▼
┌─────────────────────────────────────────────────────────────────┐
│  ⚠️ 人工确认点 #2：MetaMask 签名                                 │
│  🔴 AI 完全无法参与——私钥在 MetaMask 里                          │
└─────────────────────────────────────────────────────────────────┘
     │
     ▼
                                                        ④ 链上执行
                                                        验证者确认
     │
     ▼
                              ⑤ AI 查询验证
                              调用 Etherscan API
                              返回：交易哈希、状态、Gas
     │
     ▼
┌─────────────────────────────────────────────────────────────────┐
│  ⚠️ 人工确认点 #3：验证交易成功                                  │
│  在 Etherscan 确认：Status: Success ✓                           │
└─────────────────────────────────────────────────────────────────┘
     │
     ▼
                              ⑥ AI 生成记录
                              保存到 Obsidian + GitHub
     │
     ▼
  完成！✅
```

## 关键边界

### AI 可以做（规划、解释、检查、记录）
- 解释交易含义
- 检查地址格式
- 预估 Gas 费
- 生成交易摘要（草稿）
- 提供操作指引
- 查询交易结果
- 生成学习记录

### AI 不能做（签名、授权、转账）
- 访问私钥
- 替用户签名
- 自动发起交易
- 修改授权额度

### 必须人工确认（3 个检查点）
1. 审查交易摘要
2. MetaMask 签名
3. 验证交易成功

## 流程说明

### 1. 解决什么问题？
帮助 Web3 新手安全完成测试网交易——AI 负责规划、解释、检查，用户负责签名、确认、验证。

### 2. AI 辅助的步骤
解释交易、检查地址、预估 Gas、生成摘要、提供指引、查询结果、生成记录

### 3. 人工确认的步骤
审查摘要、MetaMask 签名、验证成功

### 4. 如何验证结果？
通过 Etherscan 查看交易哈希、状态（Success/Failed）、区块号、确认数

### 5. 主要风险点
- AI 幻觉导致错误信息
- 地址归属无法确认
- 用户过度信任 AI
- 钓鱼攻击
- 私钥泄露

## 核心原则

```
AI 是参谋，不是司机
├── AI 规划、解释、检查
├── 人类决策、确认、执行
└── 私钥和助记词是不可触碰的红线
```

## 实际应用

这个流程就是我完成 Sepolia 转账的真实经历：
1. 告诉 Hermes 想转 0.01 ETH
2. Hermes 解释交易、检查地址、预估 Gas
3. 我在 MetaMask 里手动操作、确认、签名
4. Hermes 帮我查 Etherscan 验证结果
5. 我确认交易成功，记录哈希

**Hermes 没有碰过我的私钥，没有替我签名。**
